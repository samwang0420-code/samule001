# 🏰 StackMatrices GEO - System Architecture v1.2

## 系统概览

**核心定位**: 为Houston移民律师提供GEO (Generative Engine Optimization) 本地搜索优化的自动化引擎

**运行模式**: CLI工具 + 自动化工作流，无前端界面，专注后端数据处理能力

## 渐进式部署

### Phase 1: Demo模式 (零成本启动)

```
[CLI] → [GEO Engine] → [模拟数据] → [本地文件]
         ↓
    [评分算法]
         ↓
    [Schema输出]
```

- 无API配置
- 模拟Google Maps数据
- 本地文件存储
- 适合测试和演示

### Phase 2: Live模式 (真实数据)

```
[CLI] → [GEO Engine] → [Apify API] → [Google Maps]
         ↓
    [评分算法]
         ↓
    [本地输出]
```

- 配置 APIFY_TOKEN
- 真实Google Maps数据
- 本地文件存储
- 成本: ~$40/月

### Phase 3: Full模式 (完整系统)

```
[CLI] → [GEO Engine] → [Apify API] → [Google Maps]
         ↓              ↓
    [评分算法]    [Supabase]
         ↓              ↓
    [多路输出] ←──[持久化存储]
```

- 配置 APIFY_TOKEN + SUPABASE_KEY
- 真实数据 + 数据库存储
- 完整监控和自动化
- 成本: ~$50/月

## 核心算法

### GEO评分 (0-100)

```
总分数 = 坐标精度 + 停车便利 + Schema完整度 + 本地上下文

坐标精度 (25分)
├── 建筑级坐标: 22分
├── 街道级坐标: 15分
└── 海拔数据: +3分

停车便利 (25分)
├── 详细停车信息: 25分
├── 有停车信息: 20分
├── 评论提到停车: 10分
└── 无信息: 5分

Schema完整度 (30分)
├── 名称/地址/电话/网站/坐标/评论: 各5分

本地上下文 (20分)
├── 附近地标/法院/交通: 各3分
├── 服务区域: 5分
└── 社区信息: 3分
```

## 安全与隔离

### 数据隔离
- 客户数据通过 `client_id` 隔离
- 数据库使用 Row Level Security (RLS)
- 为未来多租户预留

### API安全
- 环境变量管理敏感信息
- 本地 `.env` 不提交到Git
- API Token分级 (开发/生产)
