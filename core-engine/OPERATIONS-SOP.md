# 📋 GEO系统运营手册 (Standard Operating Procedure)

## 一、每日定时任务

### ⏰ 任务时间表

| 时间 | 任务 | 命令 | 输出 |
|------|------|------|------|
| **06:00** | 排名监控检查 | `./scheduler.sh daily` | `logs/scheduler-YYYYMMDD.log` |
| **08:00 周一** | 周报生成 | `./scheduler.sh weekly` | 客户报告邮件 |
| **随时** | 告警检查 | `node lib/alert-system.js check [client_id]` | 即时通知 |

### 📊 Daily任务详情

**执行内容：**
```bash
for each client:
  1. 检查关键词排名变化
  2. 检测排名告警 (>3位变化)
  3. 检测竞品超越
  4. 记录到数据库
  5. 生成每日摘要
```

**获取数据：**
- ✅ 关键词当前排名 (SERP位置)
- ✅ 排名变化趋势 (vs昨日)
- ✅ 竞品位置变化
- ✅ SERP特性变化 (Local Pack/Knowledge Panel)

## 三、客户签约完整流程

### 🔄 全流程图

```
签约 → onboarding → 分析 → 优化 → 监控 → 报告
        (Day 0)   (Day 1)  (Day 3) (持续) (每周)
```

### Step 1: 签约当天 (Day 0)

**收集信息：**
- 诊所名称、完整地址
- 专业领域 (Medical Spa / Dentistry)
- 核心服务 (Botox, Fillers等)
- 联系方式、竞争对手

**创建客户档案：**
```bash
CLIENT_ID="client_$(date +%s)"
mkdir -p outputs/${CLIENT_ID}
```

---

### Step 2: Onboarding分析 (Day 1)

**执行命令：**
```bash
./medical-pipeline.js \
  "诊所名称" \
  "完整地址" \
  "专业领域" \
  "服务1" "服务2" "服务3"
```

**获取数据：**
1. **Google Maps数据** - GMB评分、评论、照片
2. **GEO分析** - GEO Score、排名预测、引用概率
3. **竞品分析** - Top 3对手、优势/弱点
4. **优化方案** - Schema代码、内容、GMB清单

**交付物：**
```
outputs/client_xxx/
├── MEDICAL-GEO-REPORT.md      # 客户报告
├── MEDICAL-DEPLOY.md          # 部署指南
├── schema.json                # Schema代码
├── location-page.html         # 位置页面
└── gmb-updates.json           # GMB更新清单
```

---

### Step 3: 实施优化 (Day 3-7)

**客户执行：**
```
Week 1 必做 (约1.5小时):
□ 部署Schema标记
□ 更新GMB描述
□ 发布3个GMB帖子
□ 添加5个Q&A
□ 提交搜索引擎索引

Week 2-4 持续:
□ 每周2-3个GMB帖子
□ 回复新评论
□ 添加客户照片
```

---

### Step 4: 持续监控 (每天自动)

**监控内容：**
```
关键词排名:
  - 抓取频率: 每日1次
  - 数据来源: Google SERP
  - 存储: Supabase + 本地

竞品监控:
  - 范围: Top 10竞品
  - 频率: 每日1次
  - 告警: 排名变化>3位

GMB数据:
  - 指标: 曝光、点击、致电
  - 频率: 每日
```

**告警规则：**
```
🔴 紧急: 排名骤降>5位、竞品到#1、GMB被改
🟡 重要: 排名变化>3位、新竞品出现、差评
🟢 日常: 排名波动<2位 (正常)
```
